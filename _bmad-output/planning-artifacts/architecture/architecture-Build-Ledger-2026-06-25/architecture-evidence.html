How Build Ledger is built

One local pipeline computes evidence from real repositories, redacts it fail-closed, and emits a single versioned file that a static page renders. Collector and page are coupled by that file and nothing else. The decisions below are recorded, not asserted — every one survived an adversarial review.

The pipeline

The load-bearing decisions

DECISION 01Where the Collector runs
The tension

The hero metric needs git history, which means cloning repos — and 35 are real private client repos. A weekly GitHub Action would put client code on a third-party runner; redaction protects the output, not the clone.

The trade

Fully-automated CI (autonomous, but client code leaves the machine and the richest modules can't run) vs local-primary (every module runs; nothing client-side ever leaves).

Chosen — Local-primary. The Collector runs only on a trusted machine; a local scheduler drives the weekly run; only the redacted file is published. Client code never leaves my machine — a privacy posture that is itself judgment on display.
DECISION 02How the page consumes the data
The tension

The page is a graded work-sample. A skeptic should be able to verify the numbers, not read a bundle to trust them.

The trade

Runtime fetch (drop-a-file simplicity, but a JS bundle to inspect) vs build-time render (static HTML with real numbers in view-source; a rebuild per update — free under local-primary).

Chosen — Build-time static render (Astro 6). The page is generated into static HTML/CSS/SVG; the published build-ledger.json is served beside it so a reader can diff page against file.
DECISION 03How the privacy boundary is drawn
The tension

An open collector proves the methodology a skeptic wants to audit — but the same repo must never carry private data over 35 client codebases.

The trade

Two repos with a hard air-gap (safest, but hides the methodology) vs one public repo whose safety is structural.

Chosen — One public repo. Collector + site + data are open as the work-sample; the Collector writes every private output to a path outside the repo tree, so the repo cannot contain private data by construction — not by gitignore discipline.

The invariants (15 architecture decisions)

AD-1
Local-primary runtime
Sole producer on a trusted machine; never CI; only the redacted file leaves.
AD-2
Single versioned contract
build-ledger.json is the only coupling; the page renders solely from it.
AD-3
Module ownership
Each module owns one slice; no cross-writes; absent data is explicit.
AD-4
Fail-closed redaction
Whole-document assert before publish; controlled-vocabulary free-text; a failing run does not publish.
AD-5
Private outputs out of tree
The mirror & raw evidence live outside the repo — safe by construction.
AD-6
Mirror never public
The brutally-honest view is private-only; no mirror_view key ships.
AD-7
Semantic versioning
Additive = MINOR (the v1.5 line-level path); breaking = MAJOR the page refuses.
AD-8
Build-time static render
Numbers in the HTML, not a bundle; file served beside the page.
AD-9
Aggregate-only, no transcripts
Log-derived modules emit counts/ratios only — never raw text.
AD-10
Provenance-first contract
Hero = co-authorship + artefacts; commit-share is a labelled lower bound; no spend-leaderboard structure.
AD-11
Auditable cost or none
Pinned, dated, labelled price source; never estimated — omit if unsourceable.
AD-12
Free static host, local deploy
Pre-built dist/ deploy; no CI, no host build, $0; host is swappable.
AD-13
Assembler owns collections
The entrypoint owns repositories[] & ids; modules contribute by id.
AD-14
Aggregates are derived
Totals are a pure projection of the rows — reconcilable by construction.
AD-15
Atomic, fail-closed run
Build in memory → atomic replace; tiering → aggregate → assert → write; deterministic.

Judgment on display

The whole architecture optimizes for one thing a skeptic can check: every figure is generated from source, redacted fail-closed, and traceable to the file it came from. The open collector is the methodology; the out-of-tree private drawer is the safety; the lower-bound framing of the hero number is the honesty. Nothing here asks to be trusted.
