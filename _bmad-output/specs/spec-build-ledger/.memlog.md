- (capability) CAP-2 (FR-3) Commit-level Co-Authorship Split (HERO), stated as a lower bound. intent: per repo, compute the human/agent commit split from commit authors + Co-authored-by trailers, naming each agent author, presenting the AI-co-authored share as an explicit commit-level lower bound. success: split names each agent author with counts; headline labelled a commit-level lower bound with Methodology explaining why; per-author evidence on/linked from page (counts only, no commit messages).
- (capability) CAP-3 (FR-4) Detect + classify AI-native artefacts. intent: detect CLAUDE.md/AGENTS.md/.cursorrules/.claude/mcp and assign each to exactly one of three Artefact Classes (workflow-infrastructure/delivery-artefacts/quality-controls). success: presence detected+counted per repo; each artefact under exactly one class; private repos surface aggregate signal only (no path/name).
- (capability) CAP-4 (FR-5) Aggregate agentic practice without transcripts. intent: compute aggregate practice/efficiency (cadence, model mix, cache-hit ratio, honest cost totals, cost trend) from local Claude Code + Codex logs, framed efficiency-as-craft, never reading or emitting transcript content. success: no transcript text in file or page; every cost figure carries a labelled pricing source (logged figures or pinned pricing.yml with as_of), never estimated; trend names at least one confounder; not a leaderboard.
- (capability) CAP-5 (FR-11) Sourced Retrospective in two framings. intent: produce a Retrospective from BMAD memlogs + git in two framings, a curated Window View that publishes and a brutally-honest Mirror View that is private-only. success: every claim traces to a memlog entry or git; the public file carries retrospective.window_view only and has no retrospective.mirror_view key; the Mirror View exists only in the private mirror artifact.
- (capability) CAP-6 (FR-12) In-Flight from auditable signals only. intent: compute In-Flight from live repo signals (WIP branches, open issues, draft PRs, in-code work-markers, commit trajectory), excluding any aspirational item. success: every item backed by a live repo signal at run time; private repos surface aggregate-only (no branch/issue names); section labelled live with the run timestamp.
- (capability) CAP-7 (FR-6) Safe-by-default, fail-closed redaction. intent: apply Redaction Rules every run, defaulting every private repo to display_tier aggregate_only, asserting over the whole public document before publish (fail-closed); the Allowlist (empty in v1) is the only promotion path. success: a fixture exercising all five public blocks incl. a private repo produces a published file with none of the prohibited fields (name/path/branch/message/secret/client name/transcript) and no mirror_view key; a run that cannot assert clean does not publish.
- (capability) CAP-8 (FR-8) Single build-ledger.json contract. intent: emit exactly one build-ledger.json per run conforming to the documented shape, carrying schema_version, every Module nested, a no-data Module represented explicitly (available:false). success: the file validates against the documented top-level shape + schema_version; the page renders solely from it; a missing Module is present as available:false not omitted.
- (capability) CAP-9 (FR-7) Local schedule + on-demand fast-track. intent: run the Collector weekly via a local scheduler (launchd/cron) and manually on demand, where one manual run produces a publishable cut before the schedule is enabled. success: a manual trigger produces a complete file + rebuilt page without the weekly schedule; scheduled and manual runs emit the same shape; redaction applied every run before publish.
- (capability) CAP-10 (FR-9) Publish the Generated Page with Ledger Metadata + Methodology. intent: render a build-time static page at the Route with visible Ledger Metadata, the Methodology Note, and the Excluded-from-counts Note. success: page shows run timestamp, collector version, schema_version, identities, date range, and a link to the published build-ledger.json; every headline figure is one interaction from its evidence or the Methodology.
- (capability) CAP-11 (FR-10) Survive inspection as a work sample. intent: present a page a skeptical reader can inspect without finding a fabricated figure or a leak, and without it breaking. success: no figure lacks a source + Methodology explanation; renders without error on latest stable desktop Chrome/Safari/Firefox; attempts to find private internals surface only Silhouettes.
- (constraint) Local-primary runtime (AD-1): Collector runs ONLY on a trusted local machine holding clones/logs/memlogs; sole producer of build-ledger.json + the private mirror; weekly via local scheduler, never a third-party CI runner. Private repo content/history, agent logs, and the GitHub token never leave the machine; only the redacted build-ledger.json does.
- (constraint) One coupling, semver contract (AD-2/AD-7): build-ledger.json is the only interface between Collector and Page; the Page renders solely from it; schema_version is semver, additive=MINOR (forward-compat attribution rep for v1.5 line-level), breaking=MAJOR (page refuses an unsupported MAJOR).
- (constraint) Module ownership + atomic assembly (AD-3/13/14/15): each Module owns one slice; the assembler (collect.py) owns repositories[] + the stable ids; aggregates are a derived projection of the rows; a run builds the whole doc in memory then atomic-replaces; order = tiering -> contributions -> aggregation -> whole-doc redaction assert -> write; degraded Module = available:false, fatal error aborts publish; deterministic output.
- (constraint) Fail-closed whole-document redaction (AD-4): every private repo defaults to display_tier aggregate_only (Silhouette); a single assert runs over the ENTIRE public document before publish; free-text fields are controlled-vocabulary by construction; the run fails closed if the assert fails; display_tier is a fixed 3-value enum (public|redacted|aggregate_only); Allowlist is the only promotion path and ships empty in v1.
- (constraint) Structural privacy + mirror never public (AD-5/AD-6): private outputs (mirror.json, raw CSVs) are written OUTSIDE the repo tree so the one public repo cannot contain private data by construction; the public retrospective carries window_view only and ships no mirror_view key. Safety-critical over ~35 real client repos; rules reviewed before any private clone.
- (constraint) Build-time static, free, local-deploy (AD-8/AD-12): the Page is static HTML/CSS/SVG built from the file (no runtime fetch), deployed PRE-BUILT to a free static host from local (no Actions, $0); host swappable (default Cloudflare Pages). Route = raedmund.com/engineering.
- (constraint) Aggregate-only telemetry + provenance-first anti-vanity (AD-9/10/11): log-derived modules emit counts/ratios only, never transcript text; hero = co-authorship + AI-native artefacts, commit-share a labelled lower bound; the contract carries NO spend-leaderboard/Wrapped/live-counter structure; cost is auditable (pinned/labelled) or omitted, never estimated.
